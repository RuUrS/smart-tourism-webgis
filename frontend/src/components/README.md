# components

`components/` 目录用于存放前端 Vue 组件。本项目的主要页面功能、地图展示、登录入口、账号状态栏等界面模块均在该目录中实现。

## 文件说明

| 文件                   | 作用                                                                                                                                                           |
| ---------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `MapView.vue`          | 项目前端核心地图组件，负责展示 WebGIS 地图页面，包括地图底图切换、POI 展示、在线地点搜索、天气信息、路线规划、图层控制、智能旅游助手、数据大屏入口等主要功能。 |
| `AmapEnhancedView.vue` | 高德地图增强视图组件，用于高德地图相关功能的扩展展示，可用于在线地图能力测试、高德地图服务集成或增强地图视图展示。                                             |
| `AuthEntry.vue`        | 系统入口组件，用于用户进入系统前的身份选择，支持游客进入、用户登录和用户注册。游客可直接进入地图系统，登录用户可保存个人使用记录。                             |
| `AuthPanel.vue`        | 账号状态栏组件，显示当前用户状态。未登录时显示游客模式、登录、注册；登录后显示用户名、我的记录、退出登录等操作入口。                                           |
| `HelloWorld.vue`       | Vue 默认示例组件或早期测试组件，主要用于项目初始化阶段的页面测试，如后续项目中未使用，可考虑保留作示例或删除。                                                 |

## 组件关系说明

- `App.vue` 负责控制系统入口和主页面展示。
- `AuthEntry.vue` 用于进入系统前的登录、注册、游客入口。
- 用户进入系统后，主要展示 `MapView.vue` 地图页面。
- `AuthPanel.vue` 作为账号状态栏叠加在地图页面上，用于登录状态展示和用户记录查看。
- `MapView.vue` 会调用后端 `/api` 接口，实现 POI 查询、路线规划、天气查询、智能助手等功能。

## 说明

- 本目录只存放前端 Vue 组件代码。
- 前端接口请求应尽量使用 `/api/...` 路径，通过 Vite 代理转发到后端，避免在组件中写死 `localhost` 或局域网 IP。
- 与登录状态、Token 保存、用户记录保存相关的公共方法放在 `frontend/src/utils/auth.js` 中。
- 如果某个组件不再被使用，建议先确认 `App.vue`、`router` 或其他组件中没有引用，再决定是否删除。
